Separation of Powers for Autonomous Systems

As AI systems become autonomous actors, authorization must be:

  • Explicit — produce a concrete decision artifact
  • Portable — independent of the execution platform
  • Externally Verifiable — verifiable without trusting the system performing the action
  • Action-Bound — cryptographically bound to the exact irreversible action

If these properties are missing, you do not have separation of powers.

Enforcement

$ deploy --env production

Delegate execution without delegating authority.

Agents and automation can operate with broad access. Irreversible authority remains isolated in a separate signer domain.

Actions require a production-signed receipt. No receipt → no action.

Example

An internal agent may access repositories and CI. It cannot issue a production deploy receipt. Without that receipt, production deploy cannot execute.

Approval Is Not Authority.

Code review approves changes. It does not issue a cryptographic authorization artifact.

Platform state satisfies rules. It does not produce a portable receipt.

In GitHub, approval satisfies policy inside the system. In Permission Protocol, approval generates a signed capability that must be redeemed at execution.

Authorization must be explicit and externally verifiable.
