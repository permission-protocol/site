Authorization Requirements for Autonomous Systems

Explicit

Authorization produces a concrete decision artifact, not implicit platform state.

Portable

The artifact is independent of any execution platform or runtime context.

Externally Verifiable

Verification does not require trusting the system performing the action.

Action-Bound

The artifact is cryptographically bound to the exact irreversible action.

When Authority Lives Inside the Execution Plane

  • Approval state can be modified with configuration changes
  • Execution system implicitly trusts its own authorization state
  • Deploy and authorization share a trust boundary
  • Post-hoc logs substitute for explicit authorization artifacts

This is not separation of powers.

Without External Authority

PR approved → merge → deploy auto-triggers

Authorization state stored as metadata

With Permission Protocol

PR approved → receipt issued → receipt redeemed at deploy

Execution requires cryptographic authorization artifact

Enforcement

Deploy cannot execute without redeeming a production-signed receipt.

Deploy Gate

Receipt verification step in your CI pipeline.

- name: Verify Authorization
  run: |
    pp verify \
      --scope "${{ github.repository }}" \
      --commit "${{ github.sha }}" \
      --env production
    
- name: Deploy
  if: success()
  run: ./deploy.sh

If receipt verification fails, deploy exits non-zero.
