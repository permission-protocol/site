Authorization Requirements for Autonomous Systems

Explicit

Authorization produces a concrete decision artifact, not implicit platform state.

Portable

The artifact is independent of any execution platform or runtime context.

Externally Verifiable

Verification does not require trusting the system performing the action.

Action-Bound

The artifact is cryptographically bound to the exact irreversible action.

Enforcement

Deploy cannot execute without redeeming a production-signed receipt.

Does This Apply to Your Stack?

  • AI systems generate or modify production-bound code
  • Deploy is automated through CI/CD
  • Agents or bots hold execution privileges
  • Production changes are irreversible

If yes, authorization must be external.

Permission Protocol

Deploy Gate

Receipt verification step in your CI pipeline.

Agent
↓
GitHub
↓
CI
↓
Permission Protocol
↓
Production
- name: Verify Authorization
  run: |
    pp verify \
      --scope "${{ github.repository }}" \
      --commit "${{ github.sha }}" \
      --env production
    
- name: Deploy
  if: success()
  run: ./deploy.sh

If receipt verification fails, deploy exits non-zero.
