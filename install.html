Install Deploy Gate

Add fail-closed enforcement to any GitHub repo. No receipt → no deploy. Three steps, one workflow file.

Prerequisites

  • A GitHub repository with a CI/CD workflow
  • A Permission Protocol account (request access if you don't have one)
  • Your PP API key and Tenant ID (from your dashboard)

Add your API key to GitHub Secrets

In your repo, go to Settings → Secrets and variables → Actions and add two repository secrets:

# Secret 1 Name: PP_API_KEY Value: <your API key from the PP dashboard> # Secret 2 Name: PP_TENANT_ID Value: <your tenant ID from the PP dashboard>

Add the Deploy Gate workflow

Create .github/workflows/deploy-gate.yml in your repo:

name: Deploy Gate on: pull_request: types: [opened, synchronize, reopened] jobs: deploy-gate: runs-on: ubuntu-latest steps: - name: Verify Deploy Authorization uses: permission-protocol/verify-action@v1 with: api-key: ${{ secrets.PP_API_KEY }} tenant-id: ${{ secrets.PP_TENANT_ID }} scope: | { "scopeType": "GIT_REF_COMMIT", "ref": ${{ github.sha }}, "repo": ${{ github.repository }}, "environment": "production" }

Enable branch protection

Lock it down so the gate can't be skipped. In your repo, go to Settings → Branches → Branch protection rules:

Branch name pattern: main Enable these: ☑ Require status checks to pass before merging → Search and add: deploy-gate ☑ Require branches to be up to date ☑ Do not allow bypassing the above settings

That's it. Every PR to main now requires a valid authorization receipt before it can merge. No receipt → merge blocked → nothing reaches production.

Verify it works

Open a test PR against main. You should see the Deploy Gate check appear as "pending" — it will block until a valid receipt is issued through Permission Protocol.

To approve a deploy, create a deploy request in your PP dashboard. Once approved and the receipt is issued, the check turns green and the PR can merge.
