Protect This Repo

No merge without approval. 2 minutes. Zero dependencies.

1

Get Your API Key

~1 minute
  1. Go to app.permissionprotocol.com
  2. Sign up or log in
  3. Go to Settings → API Keys → Create Key
  4. Name it deploy-gate
  5. Scopes: receipts.verify, deployRequests.create
  6. Copy the key (shown only once)
🔒 Least privilege: This key can create approval requests and verify receipts. It cannot approve requests or access other tenants.
2

Add Secret to Your Repo

~30 seconds
gh secret set PP_API_KEY -b "pp_live_your_key_here"

Or: Repo → Settings → Secrets → Actions → New repository secret

3

Add the Workflow

~30 seconds

Create .github/workflows/deploy-gate.yml in your repo:

.github/workflows/deploy-gate.yml name: Deploy Gate on: pull_request: branches: [main] jobs: gate: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: permission-protocol/deploy-gate@v1 with: pp-api-key: ${{ secrets.PP_API_KEY }}

Commit and push to main.

4

Test It

~30 seconds
git checkout -b test-gate echo "# test" >> .github/workflows/test.yml git add . && git commit -m "test: trigger deploy gate" git push origin test-gate

Open a PR to main. The check will fail immediately:

═══════════════════════════════════════════
  🔐 DEPLOY AUTHORIZATION REQUIRED

  ❌ NO RECEIPT — Human approval required

  This PR changes protected deployment files.
  A human must approve before merge.

  👉 APPROVE: https://app.permissionprotocol.com/pp/deploy-requests/dr_...
═══════════════════════════════════════════

Click the approval link. Approve. Re-run the workflow.

  ✅ RECEIPT VERIFIED — Deploy authorized
  Receipt: rcpt_dg_... | Approved by: you@company.com

✅ You're protected.

No PR touching protected paths can merge without a signed receipt.

Common Questions

What paths are protected by default?
deploy/* and .github/workflows/*. Customize with the protected-paths input:

protected-paths: '^(src/critical/|infra/|terraform/)'
What if the check fails unexpectedly?
Check that your PP_API_KEY secret is set correctly (starts with pp_live_ or pp_test_) and has the right scopes. See the troubleshooting guide.
Can I use this with branch protection rules?
Yes — and you should. Go to Settings → Branches → Branch protection rules and add "Deploy Gate" as a required status check. This makes the gate mandatory.
How is the receipt signed?
Ed25519 signature over a canonical JSON payload. Each receipt is scoped to a specific repo, commit SHA, and environment. Receipts expire. See Receipt Model for the full spec.
Is it free?
Deploy Gate is open source. During early access, the hosted approval service is free. We'll announce pricing well before any changes.
